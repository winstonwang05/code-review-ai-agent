# searchSnippets 降级处理实现总结

## 概述
`searchSnippets` 方法实现了完整的降级处理机制，确保在向量检索失败时能够自动降级到 BM25 检索，保证系统的可用性。

## 实现逻辑

### 1. 主要处理流程

```java
public List<String> searchSnippets(String query, int topK) {
    // 1. 尝试向量检索
    List<Document> vectorResults = tryVectorSearch(query, topK);

    // 2. 尝试 BM25 检索
    List<Integer> bm25ChunkIndices = tryBM25Search(query, topK);

    // 3. 判断情况并处理
    if (bothEmpty) return emptyList;
    if (vectorOnly) -> 使用向量结果
    if (bm25Only) -> 使用 BM25 降级
    if (both) -> RRF 融合
}
```

### 2. 降级策略

#### Case 1: 向量检索成功，BM25 失败
- 使用纯向量检索结果
- 格式：`【标题】\n内容` 或 `内容`

#### Case 2: 向量检索失败，BM25 成功 (降级模式)
- 使用 BM25 检索的 chunks
- 按 RRFRank 分数排序
- 添加 `[BM25]` 前缀标识
- 确保去重

#### Case 3: 双路召回成功 (RRF 融合)
- 使用 RRF (Reciprocal Rank Fusion) 算法融合
- 添加来源标识：
  - `[向量]` - 仅来自向量检索
  - `[BM25]` - 仅来自 BM25 检索
  - `[双路]` - 两个检索都包含此 chunk
- 按融合分数排序

### 3. 关键实现细节

#### Chunk 去重机制
- 使用 `Set<String> processedChunks` 避免重复处理相同内容
- 基于内容而非索引进行去重，确保语义一致性

#### RRF 融合算法
```java
// RRF 分数计算
score += 1.0 / (k + rank + 1)

// k = 60 经验参数
// rank = 检索排名 (0-based)
```

#### 结果标识
- 提供详细的来源信息，便于调试和分析
- 记录各类结果的分布情况

### 4. 错误处理

#### 向量检索异常
```java
try {
    vectorResults = vectorStore.similaritySearch(...);
} catch (Exception e) {
    log.warn("【降级】向量检索异常或超时，不中断流程: {}", e.getMessage());
    // 继续执行 BM25 检索
}
```

#### BM25 检索异常
```java
try {
    bm25ChunkIndices = searchBM25(query, topK);
} catch (Exception e) {
    log.warn("【降级】BM25 检索异常，不中断流程: {}", e.getMessage());
    // 继续返回空列表（如果向量也失败）
}
```

### 5. 性能优化

1. **提前终止**：一旦收集到足够的结果，立即停止处理
2. **内存优化**：使用轻量级的数据结构存储中间结果
3. **日志优化**：只在必要时记录详细日志

## 总结

该降级处理实现确保了：

1. **高可用性**：即使向量服务不可用，BM25 仍能提供服务
2. **精确性**：返回具体的 chunks，而非随意截取的文档
3. **可观测性**：详细的日志和标识，便于问题排查
4. **性能保证**：合理的资源使用和响应时间

通过这种设计，系统在各种故障场景下都能提供可靠的检索服务。
